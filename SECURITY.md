# Security Policy / 安全策略

## Supported Versions / 支持的版本

Only the latest release build on the `master` branch is supported with security updates.

## Reporting a Vulnerability / 漏洞汇报

If you identify a security vulnerability (such as a buffer overflow in compiled network utilities, router shell privilege escalations, or remote execution bugs in custom packages), please do **not** open a public issue.

Please report vulnerabilities privately by emailing the repository owner or using GitHub's private vulnerability reporting feature.

如果您在编译的网络工具、路由器 Shell 提权或者自定义软件包中发现安全漏洞，请勿通过公开 Issue 报告，请使用 GitHub 的私有漏洞报告功能或直接发送电子邮件联系维护者。
